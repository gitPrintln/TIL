## RTO & RPO (재해복구 지표)

- 서비스 장애/사고 발생 시
- 얼마나 빨리 복구 ⟶ RTO
- 얼마나 많은 데이터까지 복구 ⟶ RPO

#### RTO (Recovery Time Objective)
```js
서비스를 언제까지 정상화해야 하는가?

시스템이 다운돼도 복구 허용 시간 범위

RTO가 2시간 → 2시간 안에 서비스 다시 살아나야 함

예) 전산장애 → 30분 내 복구 목표 = RTO=30min
```
#### RPO (Recovery Point Objective)
```js
데이터를 얼마나 이전 시점까지 복원할 수 있나?

백업 주기와 직접 연결됨

RPO가 10분이라면 장애 발생해도 최근 10분 전 데이터까진 복구되어야 함

예) 1시간마다 백업 = 장애 시 최대 1시간치 데이터 손실 → RPO=1h
```

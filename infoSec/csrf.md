## CSRF (Cross-Site Request Forgery)
```js
핵심 개념
로그인된 사용자를 속여서 “의도하지 않은 요청”을 보내게 하는 공격

어떻게 공격되나?
사용자가 사이트 A에 로그인 (세션 유지)
공격자가 만든 악성 사이트 B 접속
사이트 B가 A로 요청 전송
브라우저는 세션 쿠키 자동 포함 → 요청 성공
```
- 왜 위험한가?
```js
사용자는 아무것도 안 했는데
비밀번호 변경, 송금, 설정 변경 발생 가능
```
- 방어 방법
```js
✔ CSRF Token
✔ SameSite 쿠키 옵션
✔ Referer / Origin 검사
✔ 중요 요청은 POST + 재인증
```

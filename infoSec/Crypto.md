## 암호 기법 – 대칭키 vs 비대칭키

#### 1) 대칭키 암호화 (Symmetric Encryption)
- 하나의 키로 암호화 + 복호화

```js
특징

빠름

구현 간단

키가 노출되면 끝이라 키 관리가 가장 큰 문제

예시 알고리즘

AES, DES, 3DES

예시 상황

서버 내부 데이터 암호화

HTTPS 통신 중 실제 데이터 암호화
```

#### 2) 비대칭키 암호화 (Asymmetric Encryption)
- 공개키(Public Key)와 개인키(Private Key) 2개 사용
- 공개키 → 암호화

- 개인키 → 복호화(또는 그 반대 조합도 사용 가능)

```js
특징

키 교환이 안전

속도는 느림

인증/전자서명에 사용

대표 알고리즘

RSA, ECC

예시 상황

HTTPS 인증서

전자서명

세션 키 교환
```
